¿Para qué vas a pagar una suscripción mensual a una API de scraping si puedes montar tu propio sistema de extracción multinivel que consume casi cero recursos?

Para alimentar PunchCard (nuestro 'Transfermarkt' del boxeo amateur), extraemos datos de decenas de federaciones sin API pública. Al principio, la tentación de levantar navegadores headless para todo era grande. Pero en un servidor de 16GB de RAM, la eficiencia manda.

Diseñamos una jerarquía de scraping progresiva: Direct HTTP ➔ SearXNG ➔ Firecrawl ➔ Playwright. Empezamos siempre por lo más ligero y barato:

1. El 80% de los datos se extrae con peticiones HTTP directas. Es instantáneo, no consume RAM y es indestructible si la estructura no cambia. Levantar un navegador para esto es matar moscas a cañonazos.
2. Si la web depende de JavaScript básico, delegamos en SearXNG o Firecrawl para renderizar lo justo sin penalizar el rendimiento.
3. Solo cuando hay que interactuar (saltar clics, rellenar formularios o inicios de sesión complejos) desplegamos la artillería pesada con Playwright.

Esta arquitectura nos permite indexar más de 11.000 boxeadores y 19.000 combates sin depender de APIs de pago y manteniendo el consumo de memoria al mínimo.

La lección: antes de añadir otra suscripción de pago a tu presupuesto, ordena tus herramientas de menor a mayor coste de computación. La fuerza bruta casi nunca es la opción más inteligente.
