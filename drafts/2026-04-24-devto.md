## The Setup Nobody Told Me Was Possible

Five months ago, I started building a multi-agent AI system. Every tutorial assumed cloud infrastructure, Kubernetes clusters, and budgets measured in hundreds of dollars per month.

I had a Windows Server, 16GB of RAM, a GTX 1080, and stubbornness.

Today, 21 specialized AI agents handle everything from writing blog posts to managing social media to powering a legal document search engine with 440K documents. Monthly API cost: **$5.43**.

Here's everything I learned — and everything that broke along the way.

## Architecture

```
                    ┌─────────────┐
                    │  Telegram    │
                    │  (control)   │
                    └──────┬──────┘
                           │
┌──────────┐        ┌──────▼──────┐        ┌──────────┐
│  Crons   │───────>│    Main     │<───────│  User    │
│ (50+)    │        │   Agent     │        │  (DM)    │
└──────────┘        └──────┬──────┘        └──────────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
    ┌─────▼─────┐   ┌─────▼─────┐   ┌─────▼─────┐
    │  Thinker  │   │ Dev Agents│   │  Social   │
    │  (Pro)    │   │  (Flash)  │   │  (Flash)  │
    └───────────┘   └───────────┘   └───────────┘
```

**Key principle:** The main agent is the only orchestrator. Sub-agents can't call other sub-agents. No chains, no recursive spawning, no surprise $3 bills from runaway agent conversations.

Routing is IF/ELSE, not LLM judgment. When someone asks about code, it goes to the dev agent. When they ask about research, it goes to the research agent. Simple pattern matching beats "smart" routing every time at this scale.

## The Cost Journey

| Month | Cost | What Changed |
|-------|------|-------------|
| 1 | $45 | Everything on Gemini Pro |
| 2 | $28 | Moved social + content agents to Flash |
| 3 | $15 | Sub-agent model Pro → Flash (biggest single saving) |
| 4 | $9 | Silent crons, local embeddings via Ollama |
| 5 | $5.43 | Fine-tuned prompt lengths, disabled chatty memory crons |

**The revelation:** Gemini Flash handles ~90% of agentic tasks at ~90% quality for ~10% of the cost. The 10% where Pro matters (complex reasoning, legal analysis) is real — but most agent work is routing, formatting, summarizing, and posting.

## What Actually Runs

### Blog Pipeline ($0.05/day)
Three-stage cron system, Mon/Wed/Fri:
1. **08:30 — Research:** Agent searches current news (MITECO, EU regulations, BOE), picks topic following rotation rules, writes 800-word SEO article in HTML, generates featured image, uploads to WordPress as draft
2. **09:30 — Review:** Different agent reviews the draft against 12 quality checks (internal links, FAQ section, markdown contamination, CTA formatting). Blocks publication if critical issues found
3. **10:30 — Publish:** If review passed, publishes the draft and updates the knowledge base to prevent topic repetition

### Social Automation ($0.15/month)
- 120+ tweets/month across accounts with distinct AI personalities
- Playwright-based posting (no Twitter API needed)
- Cookie-based auth with automatic session management

### Legal Document Platform
- 440K government documents ingested
- 6.9M embeddings generated locally (nomic-embed-text via Ollama — free)
- Semantic search with hybrid BM25 + vector approach

### Boxing Data Platform  
- 11K fighters, 19K matches scraped and normalized
- ELO rating system
- Match prediction capabilities

## 5 Patterns That Saved Everything

### 1. Model Tiering
Don't use one model for everything. Route by task complexity:
- **Flash/cheap:** Formatting, posting, notifications, simple summarization
- **Pro/expensive:** Legal analysis, complex reasoning, creative writing
- **Local/free:** Embeddings, classification, simple extraction

### 2. Silent Infrastructure
My Telegram went from 300 messages/day to 20. The fix: cron jobs should only notify on errors. Nobody needs "heartbeat OK" 48 times a day.

### 3. Central Orchestrator Pattern
Sub-agents calling sub-agents is a cost bomb. One orchestrator routes everything. Period.

### 4. NSSM + AppThrottle on Windows
Every Windows service gets `AppThrottle: 30000` (30 seconds between restart attempts). Without this, a crashed service restarts instantly in an infinite loop, eating CPU and generating thousands of error logs per minute.

### 5. Knowledge State Files
Each pipeline maintains state files (published articles, image hashes, review results). This prevents duplicate work and enables the "continue where you left off" pattern that makes autonomous operation possible.

## The Disasters

1. **Sub-agent chains:** An agent called another agent, which called another. $3 in tokens in under 2 minutes. Fix: blocked `sessions_spawn` for all sub-agents
2. **Memory dreaming:** A cron that made agents "reflect on their memories" ran every 30 minutes. Cost more than all 21 agents combined. Fix: limit 10 memories, 30-day window
3. **WordPress URL:** WP installed at `/blog/` means the API is at `/blog/wp-json/`, not `/wp-json/`. Cost us 2 days of debugging silent failures
4. **Telegram flood:** 300+ messages/day. Most were "cron executed successfully". Fix: silent mode for infrastructure crons
