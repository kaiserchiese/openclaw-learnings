# LinkedIn Version - 2026-05-04

Tener 21 agentes de IA trabajando en el mismo servidor por 7 dólares al mes suena a milagro, pero el verdadero reto no es el coste: es la orquestación.

En OpenClaw hemos aprendido por las malas que si dejas que los agentes se organicen solos, acabas con un bucle infinito de "te paso la tarea" y una factura de tokens que no para de subir. 

La solución que nos ha permitido escalar sin perder la cabeza (ni la cartera) se resume en una arquitectura de jerarquía pura: el Main Agent orquesta, pero el subagente nunca llama a otro subagente.

Nuestros principios básicos de orquestación:
1. Jerarquía Centralizada: Solo el Main Agent tiene permisos para delegar. Si el experto en investigación necesita una imagen, se la pide al Main, no al generador de imágenes.
2. Enrutamiento Determinista: No dejamos que el LLM "decida" qué agente usar por instinto. Usamos reglas claras (IF/ELSE) en el prompt del sistema.
3. Model Tiering: El 90% de las tareas las resuelven modelos Flash (baratos y rápidos). Reservamos los modelos Pro solo para razonamiento complejo.

Menos caos, más control y un ahorro del 85% en costes operativos.
