Confiar en que un LLM publique de forma 100% autónoma en producción es jugar a la ruleta rusa con tu marca. Tarde o temprano, el modelo alucinará, duplicará una imagen, asignará categorías por defecto incorrectas o romperá el HTML.

En el blog de AirCO2 gestionamos un pipeline de contenidos totalmente autónomo. La clave de su fiabilidad absoluta no es usar un modelo más inteligente, sino aplicar ingeniería de sistemas clásica: "Gates Deterministas" independientes del LLM.

¿Cómo funciona esta arquitectura de calidad?

1. Gate de Research: Tras la generación, un script de validación comprueba enlaces internos, bloquea categorías no deseadas y compara el hash de la imagen destacada frente a "image_hashes.json" para no repetir recursos visuales.
2. Watchdog (Perro Guardián): Un script totalmente independiente se ejecuta asíncronamente a las 08:55. Si detecta fallos en la API o que el pipeline se ha quedado congelado, fuerza un "retrigger" automático.
3. Test Post-Publicación: Realiza una petición GET real contra la URL pública del post para confirmar un código HTTP 200 y comprobar que el cuerpo tenga un tamaño mínimo (mínimo 10KB), evitando posts vacíos.

La lección: la creatividad pertenece al dominio probabilístico del LLM; el control de calidad y la seguridad pertenecen al dominio determinista del código tradicional. Falla rápido y alerta temprano.
