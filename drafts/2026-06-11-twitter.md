¿Para qué vas a pagar una fortuna al mes en APIs de scraping si puedes montar un stack de extracción multinivel que te cuesta casi cero? En OpenClaw procesamos webs complejas con un servidor de 16GB.
---
El secreto para no quemar tokens ni RAM es nuestra jerarquía de scraping: Direct HTTP -> SearXNG -> Firecrawl -> Playwright. Empezamos con lo más ligero y barato, y solo escalamos cuando la web se pone tonta.
---
El 80% de las páginas de federaciones deportivas que procesamos se solucionan con una simple petición HTTP. No hace falta levantar un navegador headless para leer una tabla HTML de toda la vida, por mucho que nos guste complicarnos la vida.
---
¿Que la web requiere JavaScript o protección básica? Pasamos a SearXNG o Firecrawl. ¿Que se pone difícil o requiere interacciones como logearse? Desplegamos Playwright. Así optimizamos el uso de RAM y CPU de forma lógica.
